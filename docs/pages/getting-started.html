Getting Started with Linear CLI

In five minutes you'll be managing your Linear workspace from the terminal. The CLI outputs JSON by default for AI agents, or add --human for readable tables. Let's get you set up.

Installation

Prerequisites

  • Go 1.21+ (for building from source)
  • A Linear account with API access

Option 1: Go Install (Recommended)

If you have Go installed, this is the fastest way:

go install github.com/juanbermudez/linear-cli/cmd/linear@latest

Option 2: Download Binary

Download the pre-built binary for your platform:

# macOS/Linux
curl -sSL https://github.com/juanbermudez/linear-cli/releases/latest/download/linear-$(uname -s)-$(uname -m) -o /usr/local/bin/linear
chmod +x /usr/local/bin/linear

# Verify installation
linear --version

Option 3: Build from Source

git clone https://github.com/juanbermudez/linear-cli.git
cd linear-cli
make build
make install

Authentication

Get Your API Key

  1. Go to Linear
  2. Click your avatar → Settings
  3. Navigate to API → Personal API keys
  4. Create a new key (it starts with lin_api_)

Option 1: Interactive Setup (Recommended)

The setup wizard guides you through authentication and team selection:

linear config setup

Option 2: Direct Login

Authenticate directly with your API key:

# Login and set default team
linear auth login --api-key lin_api_xxxxx --team ENG

# Or just login (set team later)
linear auth login --api-key lin_api_xxxxx

Option 3: Environment Variable

For CI/CD or scripting, use the environment variable:

export LINEAR_API_KEY=lin_api_xxxxx
linear whoami

Option 4: Stdin (for AI Agents)

Pass the API key via stdin for non-interactive setup:

echo "lin_api_xxxxx" | linear config setup --stdin --team ENG

Credential Storage

Your API key is stored securely in your system's keychain (macOS Keychain, Windows Credential Manager, or Linux Secret Service). It never touches disk in plaintext.

Quick Start

Check Your Auth

# See who you're logged in as
linear whoami

# JSON output (default)
{
  "id": "user-123",
  "name": "Your Name",
  "email": "you@company.com",
  "admin": true
}

List Your Issues

# JSON output (default)
linear issue list --team ENG

# Human-readable table
linear issue list --team ENG --human

# Filter by state
linear issue list --team ENG --state started --human

Create an Issue

# Basic issue
linear issue create --title "Fix login bug" --team ENG

# With more details
linear issue create \
  --title "Add user authentication" \
  --description "Implement OAuth2 login flow" \
  --team ENG \
  --priority 2 \
  --label "feature"

View an Issue

# By identifier
linear issue view ENG-123

# Human-readable format
linear issue view ENG-123 --human

Search Issues

# Search across all issues
linear issue search "authentication bug"

# Limit results
linear issue search "login" --limit 5 --human

Work with Projects

# List projects
linear project list --human

# Create a project with a document
linear project create \
  --name "Q1 Auth Overhaul" \
  --team ENG \
  --with-doc \
  --doc-title "Auth Overhaul Spec"

# Add a milestone
linear project milestone create PROJECT_ID \
  --name "Phase 1: OAuth" \
  --target-date 2025-02-15

Manage Documents

# Create a document
linear document create \
  --title "API Design Doc" \
  --content "# Overview\n\nThis document describes..."

# Search documents
linear document search "authentication"

# View a document
linear document view DOC_ID --human

Configuration

Config File Location

Configuration is stored in .linear.toml in your project root or home directory:

# .linear.toml
team_key = "ENG"

Setting Configuration

# Set default team
linear config set team_key ENG

# Get a config value
linear config get team_key

# List all config
linear config list

Configuration Priority

Configuration is resolved in this order (highest to lowest priority):

  1. Command-line flags (--team ENG)
  2. Environment variables (LINEAR_TEAM_KEY)
  3. Local config file (./.linear.toml)
  4. Global config file (~/.linear.toml)

Output Modes

JSON Output (Default)

Every command outputs JSON by default, designed for AI agents and scripting:

linear issue list --team ENG
{
  "issues": {
    "nodes": [
      {
        "id": "issue-123",
        "identifier": "ENG-45",
        "title": "Fix login bug",
        "priority": 2,
        "state": {"name": "In Progress", "type": "started"}
      }
    ]
  }
}

Human-Readable Output

Add --human for formatted tables and readable output:

linear issue list --team ENG --human

ID       TITLE            PRIORITY  STATE        ASSIGNEE  UPDATED
ENG-45   Fix login bug    High      In Progress  JD        2 hours ago
ENG-44   Add OAuth        Medium    Todo         --        1 day ago

Error Handling

Errors include structured information to help AI agents self-correct:

{
  "success": false,
  "error": {
    "code": "MISSING_TEAM",
    "message": "Team is required",
    "hint": "Specify a team using --team flag or set a default team",
    "usage": [
      "linear issue list --team ENG",
      "linear config set team_key ENG"
    ]
  }
}

Common Error Codes

Code Description Resolution
NOT_AUTHENTICATED No valid API key found Run linear auth login
MISSING_TEAM Team required but not specified Add --team or set default
NOT_FOUND Resource doesn't exist Check the ID/identifier
VALIDATION_ERROR Invalid input Check required fields

Caching

The CLI caches frequently-accessed data to reduce API calls:

Resource TTL Refresh Command
Workflow States 24 hours linear workflow cache --team ENG
Project Statuses 24 hours linear status cache
Users 24 hours linear user list --refresh
Labels 24 hours linear label list --refresh

Cache Location

Cache files are stored in ~/.cache/linear-cli/:

~/.cache/linear-cli/
  workflows-team-ENG.json
  statuses-workspace.json
  users-workspace.json
  labels-team-ENG.json

Clear Cache

# Clear all cache
rm -rf ~/.cache/linear-cli/

# Or refresh specific resources
linear workflow cache --team ENG
