Install in Seconds

One command to install. One command to authenticate. You're ready.

1

Install the CLI

# Using Go
go install github.com/juanbermudez/agent-linear-cli/cmd/linear@latest

# Or download binary from releases
curl -sSL https://github.com/juanbermudez/agent-linear-cli/releases/latest/download/linear-$(uname -s)-$(uname -m) -o /usr/local/bin/linear
chmod +x /usr/local/bin/linear
2

Authenticate

# Interactive setup (recommended)
linear config setup

# Or login directly with API key
linear auth login --api-key lin_api_xxxxx --team ENG
3

Start Using

# List your issues
linear issue list --human

# Create an issue
linear issue create --title "Fix login bug" --team ENG

# View issue details
linear issue view ENG-123

Frequently Asked Questions

Why JSON-first output?

This CLI is designed for AI agents that need structured, parseable output. JSON provides consistent schemas that agents can rely on. Every response includes success status, data, and helpful error hints. Add --human for readable tables when you're at the terminal.

How are credentials stored?

Your Linear API key is stored in your system's secure keychain (macOS Keychain, Windows Credential Manager, or Linux Secret Service). It never touches disk in plaintext. You can also use the LINEAR_API_KEY environment variable or pass --api-key directly.

What about caching?

The CLI caches workflows, statuses, users, and labels for 24 hours to reduce API calls. Use linear workflow cache or linear status cache to force a refresh. Cache files are stored in ~/.cache/linear-cli/.

Can I use this with Claude Code or other AI tools?

Absolutely. That's the primary use case. The JSON output with consistent schemas, helpful error hints, and usage examples makes it ideal for AI agents. Claude Code can call these commands directly and parse the responses reliably.

Where do I get a Linear API key?

Go to Linear → Settings → API → Personal API keys. Create a new key with the permissions you need. The key starts with lin_api_. Keep it secret!
